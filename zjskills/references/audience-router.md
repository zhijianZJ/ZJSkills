# Audience Router

Choose the route from the result the user wants, not merely from their current title. A founder may be seeking employment; an employee may be testing a side business.

## AI employment or career transition

Use the full employment diagnosis when the user wants an AI job, a career change, a target role, target companies, employability evidence, a job-oriented learning route, resume or interview preparation, or a decision about training for employment.

Typical subgoals:

- Move into an AI technical role.
- Move into an AI product, solution, operations, sales, or delivery role.
- Combine an existing industry background with AI.
- Decide whether to transition directly or first build AI evidence in the current role.

## Current-role AI improvement

Use a light diagnosis when the user wants better quality, speed, decision-making, automation, or influence in the current role without a near-term job transition.

Identify one repeated real workflow, its current cost or failure, the AI-supported change, and a before/after measure. Treat AI tool use as a cross-role capability unless current hiring evidence supports a standalone target role.

## AI side-income work

Use a light diagnosis when the user wants bounded income outside their main job. Route toward the overlap of existing expertise, a specific buyer, and a repeatable deliverable.

Possible forms include AI-assisted content, research, automation implementation, training, consulting, design, development, or industry-specific services. Do not promise revenue. Require one small demand test before a large build or purchase.

## AI entrepreneurship

Use a light diagnosis when the user wants to build a company, product, or business line. Distinguish:

- AI product: build a software product or technical capability.
- AI service: sell a defined outcome using AI-enabled delivery.
- Existing-business upgrade: improve acquisition, delivery, operations, or cost in an existing business.

Start with customer problem, buyer, current alternative, founder advantage, and minimum validation. Do not generate an elaborate strategy without evidence.

## Learning or exploration

Use a light diagnosis when the user mainly wants AI literacy, tool familiarity, project experience, or technical depth without a defined employment or business outcome.

Choose one initial result:

- Explain core concepts accurately.
- Improve one real workflow.
- Build one bounded project.
- Test readiness for deeper technical study.

## Ambiguous or multiple goals

If the user has multiple goals, ask which result matters most in the next 6–12 months only when this changes the route. Otherwise select a primary route, state the choice, and treat the other goals as secondary constraints.
